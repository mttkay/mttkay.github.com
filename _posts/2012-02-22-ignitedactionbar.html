Changes to ActionBarCompat:

  • Completely reusable drop-in solution
  • Configurable via ActionBarConfig
  • Better refresh spinner logic:
    • Handles pending updates to refresh spinner
    • Keeps track of active task count
    • action view is an animated image view that uses the same icon rather than a ProgressBar
  • Support for enabling/disabling home button click
  • Simpler API
    • no need to implement getMenuInflater()
    • no need to inflate menu manually
  • Support for adding custom views

Limitations:

  • no support for advanced attributes such as ActionProvider
  • no "ifRoom" etc.
